## 1. 系統架構與設計模式

本系統秉持 **SOLID** 物件導向設計原則，採用分層架構進行開發，以確保高內聚、低耦合與未來的可擴展性。

```text
┌─────────────────────────────────────────────────────────┐
│                    智慧網頁控制面板 (Web UI)             │
│        HTML5 / CSS3 (毛玻璃暗黑樣式) / Vanilla JS       │
└────────────────────────────┬────────────────────────────┘
                             │ JSON API
┌────────────────────────────▼────────────────────────────┐
│                    Flask API 伺服器 (app.py)             │
└────────────────────────────┬────────────────────────────┘
                             │
┌────────────────────────────▼────────────────────────────┐
│                 結帳引擎入口 (engine.py Core)            │
└────────────────────────────┬────────────────────────────┘
                             │
            ┌────────────────┴────────────────┐
            ▼                                 ▼
┌───────────────────────────┐     ┌───────────────────────────┐
│  強健解析器 (parser.py)   │     │   領域模型 (models.py)    │
│  特徵簽名比對與註解剝離   │     │  Product, CartItem, Cart, │
│                           │     │  Promotion, Coupon        │
└───────────────────────────┘     └───────────────────────────┘
```

### 核心設計模式：策略模式 (Strategy Pattern)
* 在電商結帳系統中，促銷規則與折價券套用邏輯千變萬化。若將這些邏輯硬編碼在 `Cart` 或結帳函數中，會導致程式碼難以維護，且違反「開放封閉原則 (OCP)」。
* 本專案將 **品類促銷折扣 (`Promotion`)** 與 **滿額折抵優惠券 (`Coupon`)** 分別抽象為獨立的策略物件。
* `Cart` 核心物件僅需維護這兩組策略對象的計算合約，使得未來若要擴展新的促銷類型（如「買一送一」、「滿件折」等），僅需繼承或新增相應的策略類別即可，核心結帳邏輯完全無需更動。

## 2. 程式碼審查報告 (Code Review)

### A. 領域模型層 (`models.py`)
* **優點**：
  * 商品目錄採用靜態字典定義與大小寫不敏感（Case-insensitive）自動對齊機制，即使外部輸入不規則空格或大小寫，也能安全將商品歸入「電子、食品、日用品、酒類」等預設品類。
  * `CartItem` 與 `Cart` 職責劃分清晰。`CartItem` 封裝單項商品的數量乘積精度；`Cart` 封裝品類折扣的策略尋優邏輯。
* **改進點 (Refinement Note)**：
  * 目前當促銷重疊時，系統採用最優策略（`min(p.discount_rate)` 折扣率最低即折扣最深者）。未來若促銷規則允許疊加，可擴展 `Promotion` 的套用合約。

### B. 解析器層 (`parser.py`)
* **優點**：
  * 採用了創新的 **「特徵簽名比對法」**。該方法不依賴傳統的區塊順序，而是藉由每行字串的特徵（如 `|` 字元判定促銷、`*` 與 `:` 判定商品明細、空格分隔的 3 個部分判定優惠券）動態對齊。
  * 具備註解一鍵剝離功能（自動識別 `//` 並切除），能完美排除註解、前後空格及空行的干擾，具備極高的容錯能力。

### C. 測試覆蓋 (`test_engine.py`)
* **優點**：
  * 嚴格落實測試驅動開發 (TDD) 的單元測試規範。
  * 測試案例不僅涵蓋了 Case A 和 Case B 的基準值驗證，亦深入涵蓋了「過期優惠券」、「未達優惠券門檻」、「高精度邊界四捨五入（銀行家進位點）」與「不規則排版容錯」等極限案例。

## 3. 核心模組功能介紹

* **`Product`**：存放商品名稱、單價與品類標籤。
* **`CartItem`**：表示購買的特定商品對象與數量。
* **`Promotion`**：保存品類、促銷折扣率與促銷生效日期。
* **`Coupon`**：保存折價券到期日、滿額折抵門檻與折抵金額。
* **`Cart`**：購物車主體，計算品類折扣及最終結算。
* **`CartInputParser`**：執行強健字串清洗、剖析日期並轉換成 Python 的領域實體。

## 4. 高精度金額運算說明

本專案拒絕使用標準浮點數 (`float`)，全面採用 Python 商業級高精度 **`decimal.Decimal`** 進行所有金額計算。

### 為什麼這在電子商務中至關重要？
在 IEEE 754 浮點數標準下，電腦使用二進位來逼近十進位小數，這會導致微小的精度誤差：
```python
# 標準 Python 浮點數運算
0.1 + 0.2  # 輸出: 0.30000000000000004
```
在商品累加與打折運算中，若使用浮點數，這些微小誤差會不斷累積，最終導致結帳金額與發票不符。

### 解決方案與四捨五入
1. **過程高精度**：在整個商品累計與折扣折扣計算過程中，所有數值均保持 `Decimal` 的完整原始精度。
2. **最終進位**：僅在結帳的最後一步（即調用 `Cart.checkout()` 輸出時），才使用 `.quantize(Decimal("0.01"), rounding=ROUND_HALF_UP)` 將總付金額精確保留至小數點後兩位，實行標準商業四捨五入。

## 6. 本地快速啟動與驗證指南

您可以引導 WisdomGarden 的技術專家透過以下三種方式驗證成果：

### 方式一：執行自動化測試
```bash
python -m unittest discover -s tests
```

### 方式二：命令行 CLI 檔案結帳
```bash
# 測試 Case A 檔案
python main.py tests/case_a.txt

# 測試 Case B 檔案
python main.py tests/case_b.txt
```

### 方式三：啟動全端網頁控制面板 (視覺亮點)
1. 啟動後端伺服器：
   ```bash
   python app.py
   ```
2. 開啟瀏覽器訪問 `http://127.0.0.1:5000`。
3. 您可以透過介面快捷按鈕點選 **「套用 Case A」** 或 **「套用 Case B」** 載入範本，點擊 **「開始結算」**，右側即會以流暢的動態出單動畫顯示精確度至小數點後兩位的視覺化結帳收據，完整拆解各個品類折扣與折價券套用明細。
